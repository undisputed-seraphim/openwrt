## Hardware Offload Tracker

The RTL8198C SoC was designed for gigabit home gateways with a rich set of
hardware offload engines.  This section tracks which are implemented, which are
in progress, and the current blockers for each.

### Done

#### L2 switching offload — `[AsicDriver]` + `[l2Driver]`

**Register:** `MSCR` (0xBB804410) bit 0 (`MSCR_EN_L2`).

Hardware MAC learning and forwarding between switch ports at wire speed.
The FDB (1024-entry 4-way hash) auto-learns source MACs; unknown destinations
are flooded per `FFCR` settings.  CPU MAC is force-programmed at row 240.

---

#### L3 routing offload — `[l2Driver]`

**Register:** `MSCR` (0xBB804410) bit 1 (`MSCR_EN_L3`).

Hardware routes between IP subnets at wire speed (~0.011ms cross-subnet).
No CPU involvement for established routes.  Requires `netif` table entries
mapping subnets to VLANs and port masks.

---

#### QoS / DSCP classification and remarking — `[l2Driver]`

**Registers:** `QOSFCR` BIT(0) (BC_withPIFG), `V4VLDSCPCR0-8` (0xBB804464+)
with `CF_IPM4DSCP_ACT_REMARK | CF_IPM4PRI_ACT_ORIG`, per-port `DSCPCR0-6`
(0xBB804734+) cleared.

Hardware classifies packets by DSCP field and remarks priority on egress.
Leaky-bucket rate limiting (`ILBPCR1`) is deferred — the PSCR register block
hangs the bus if `TRXRDY` is not yet set.  Our init order places QoS before
`SIRR_TRXRDY` in the probe, so ILB writes are skipped.

---

#### PPPoE auto-encapsulation / decapsulation — `[l2Driver]`

**Register:** `ALECR` (0xBB80440C) bit 18 (`EN_PPPOE`).

Hardware auto-encapsulates and decapsulates PPPoE headers.  Also sets `ALECR`
BIT(29) during init per the BSP `rtl865x_initAsicL2`.

---

#### 802.3x flow control (pause frames) — `[AsicDriver]`

**Register:** `MACCR` (0xBB804000) bit 14 (`INFINITE_PAUSE_FRAMES`).

Hardware generates and honors 802.3x pause frames to prevent buffer overflow.
Combined with `FCDSC` tolerance set to 48 pages (bits 4-10 = `0x30`).

---

#### Port-based VLAN decision policy — `[l2Driver]`

**Registers:** `SWTCR0` (0xBB804418) bits 17:16 = `0x01` (PORT_BASED), bit 15 =
`EnUkVIDtoCPU`; `PLITIMR` (0xBB804420), `PVCR0` (0xBB804A08).

Sets the switch to port-based VLAN mode.  Unknown-VID frames are trapped to
the CPU for software processing.  Netif entries map logical interfaces
(netifs) to physical ports via the TACI netif table.

---

#### N-way auto-negotiation (all speeds including 1000M) — `[AsicDriver]`

**Registers:** `PCRP0-4` (0xBB804104+) bits 22:18 = `0x1F` (`NwayAbility_ALL`);
PHY reg 9 BIT(9) (1000BASE-T FD advertise).  N-way restart via PHY reg 0 BIT(9).

The PCRP advertises all five speeds (1000MF, 100MF, 100MH, 10MF, 10MH) on
ports 0-4.  An explicit `mdio_write(phy, 9, v | BIT(9))` is needed because
the GPHY's 1000BASE-T control register does not automatically reflect the
PCRP NwayAbility bits.

---

#### GPHY NCTL calibration — `[AsicDriver]`

**Registers:** `phy_98c_para[]` DSP register table (19 entries of reg 27/28
pairs) uploaded per-port via the NCTL key/lock protocol on page 0xb80/0xb82.

Enables the gigabit analog signal path in the internal GPHY.  Without it the
PHY silicon does not train at 1000BASE-T regardless of N-way advertisement.
Implemented in `phy_refine()` with per-port polling and a 1000-iteration
timeout guard.

---

#### Auto-down-speed (2-pair cable detection) — `[AsicDriver]`

**Register:** `gphy_wb(0xa44, 17, ~BIT(2), BIT(2))` — extended PHY page 0xa44,
reg 17, bit 2.

When the GPHY detects only 2 pairs in the cable, it limits auto-negotiation
to 100BASE-TX.  This is correct behaviour, not a bug.  The port that
negotiates 100M with the host does so because of the physical cable.

---

#### System clock 100MHz + DSC tolerance — `[AsicDriver]`

**Registers:** `MACCR` bits 12-13 = `0x01` (CF_SYSCLK_SEL, lx_clk = 100MHz),
bits 4-10 = `0x30` (CF_FCDSC, 48-page DSC tolerance).

System clock must run at 100MHz (not the default 50MHz) for gigabit data
rates.  DSC tolerance enlarged from 24 to 48 pages to prevent drops when
flow control is triggered.

---

#### Green Ethernet — `[AsicDriver]`

Implemented via `sram98c_write()` to PHY page 0xa43.  The values match the
BSP `#ifdef ENABLE_8198C_GREEN` path (enable Green, not the `#else` disable
path).  Covers both Giga and 100M cable-length power saving.

---

#### MAC address filtering (debugfs) — `[l2Driver]`

TACI L2 table `CMD_ADD` at a fixed entry (row 32).  Write a MAC to
`/sys/kernel/debug/rtl8198c-eth/eth0/mac_filter` to drop all frames from
that source MAC at switch ingress.  Write `clear` to remove the filter.
Tested: host-to-host ping through the switch is blocked by the filter.

---

#### VLAN egress control (debugfs) — `[l2Driver]`

TACI VLAN table `CMD_ADD` at VID 8 (index 8 × 8 = entry 64, word 0).
Read/write via `/sys/kernel/debug/rtl8198c-eth/eth0/vlan_egress`.
Per-port untag bits (14:9) control which ports strip the VLAN tag on egress.

---

#### Per-port LED indicators — `[AsicDriver]`

**Registers:** `LEDCR0` (0xBB804300) = `LEDMODE_DIRECT` (bits 21:20 = `10`),
`DIRECTLCR` (0xBB804314) bits 3:0 = `0x7`.

Hardware drives the RJ45 LEDs directly to show link, speed, duplex, and activity.

---

### Partially done / blocked

#### Hardware checksum offload — `[l2Driver]` → CSCR blocked

**Target registers:** `CSCR` (0xBB804048) bits 4 = `EnL3ChkCal`, 5 = `EnL4ChkCal`.
**Descriptor fields:** `CSUM_IP` (0x0002), `CSUM_TCPUDP` (0x0001) in `ph_flags`;
`ph_ipIpv4`, `ph_ipIpv4_1st`, `ph_ipIpv6`, `ph_ipIpv6HdrLen` in pkthdr;
`ph_type` protocol codes (`PKTHDR_TCP=5`, `PKTHDR_UDP=6`).

The BSP `rtl865x_initAsicL2` enables `EnL3ChkCal|EnL4ChkCal` at init time,
and `rtl865xc_swNic.c` sets `CSUM_IP|CSUM_TCPUDP` flags per-packet when
`skb->ip_summed == CHECKSUM_PARTIAL`.  The hardware computes L3 (IP) and L4
(TCP/UDP) checksums on transmit, saving the CPU from scanning the entire
payload per packet.

**Status:** Descriptor fields, `CHECKSUM_PARTIAL` path, and `NETIF_F_HW_CSUM` /
`CHECKSUM_UNNECESSARY` flag-setting code are all written and compile-tested.
Writing ANY value to CSCR at 0xBB804048 (whether ENABLE bits, ALLOW bits,
or both) causes the board to lose all Ethernet connectivity — ping and TCP
both fail.  The address matches the BSP (`SWCORE_BASE + 0x4000 + 0x048`),
and the BSP writes the same bit pattern without issue.  Likely an init-sequence
dependency: the BSP writes CSCR after a large number of other init steps that
our driver does in a different order, and one of those steps may be a
prerequisite for CSCR to take effect correctly.

**Layer:** Should sit in `[l2Driver]` with the CSCR write in `switch_l2_init()`
and the per-packet flags in `eth_start_xmit()` under `[Kernel Interface]`.

---

#### Scatter-gather DMA — `[Kernel Interface]` → blocked (TCP hang)

**Register:** `CPUICR1` (0xB8010010) bit 6 (`CF_TX_GATHER`).  Already enabled.
**Netdev flag:** `NETIF_F_SG` — tested, breaks TCP connectivity.

The BSP's `swNic_send` chains multiple `struct mbuf` via `m_next`, with the
last fragment marked `MBUF_EOR` (bit 4).  Implementation was attempted: a
circular pool of 64 spare mbufs, per-fragment chaining with KSEG1 address
translation, matching the BSP pattern from `rtl865xc_swNic.c:1336-1372`.

**Status:** Compiles and probes without error, but enabling `NETIF_F_SG`
causes TCP connections to hang (SYN reaches board, handshake doesn't
complete).  Even with the flag removed but the SG code path present,
TCP hangs — suggesting the common TX path changes (moving `m_len`/`m_extsize`
into conditional branches) alter behaviour that the RISC depends on.
Reverted to pre-SG state.  Similar blocker pattern to CSCR: BSP code
matches but board behaviour doesn't.

**Expected benefit:** Eliminates `skb_linearize()` copies for multi-fragment
skbs.  Small (~5-10%) for standard iperf3 TCP; significant for sendfile/splice
workloads.  Prerequisite for GSO.

---

#### CF_TX_GATHER (hardware descriptor gather) — `[AsicDriver]`

**Register:** `CPUICR1` bit 6.  Already set in `switch_start()`.

Allows the RISC firmware to gather multiple descriptors into a single
transmitted packet.  Without multi-fragment skb support (`NETIF_F_SG`), this
is a no-op — every packet uses exactly one descriptor.  Harmless to leave
enabled.

---

### Not done

#### TSO/LSO (TCP Segmentation Offload / Large Send Offload) — `[Kernel Interface]`

**Registers:** `CPUICR1` bits 9:8 = `0x01` (`TX_PKTHDR_SHORTCUT_LSO` descriptor
format); packet header fields for MSS (`ph_ipIpv6HdrLen`), IP header length
(`TD_IPV4_HLEN_OFFSET`), LSO enable (`TD_LSO_MASK`).

The BSP sets `NETIF_F_TSO | NETIF_F_TSO_ECN | NETIF_F_TSO6` for non-WAN
interfaces.  Requires the SHORTCUT_LSO pkthdr format (not the default
`TX_PKTHDR_8198C_DEF` we currently use).  The 8198C BSP does NOT enable
`CONFIG_RTL_SWITCH_NEW_DESCRIPTOR`, and the `TX_PKTHDR_SHORTCUT_LSO` write
is gated by that config — meaning the BSP 8198C kernel does not use LSO.

**Blocker:** The SHORTCUT_LSO format is a different pkthdr layout.  Switching
to it would require understanding the descriptor field changes and verifying
the RISC firmware is compatible.  As the BSP itself does not use LSO for
8198C, this is non-essential for feature parity with the reference driver.

---

#### GSO (Generic Segmentation Offload) — `[Kernel Interface]`

**Netdev flag:** `NETIF_F_GSO`.  Requires scatter-gather (`NETIF_F_SG`) as a
prerequisite.  The BSP 8198C config has `CONFIG_RTL_GSO=y`.

GSO allows the kernel to defer segmentation of large packets until the last
moment before the hardware queue.  Without it, large send operations are
segmented in the network stack, consuming CPU.  Depends on SG working first.

---

#### Hardware NAT (HNAT) — `[l3Driver]`

**BSP subsystem:** `l3Driver/rtl865x_hwNAT.c`, `rtl_nic.c` lines 613-617,
`CONFIG_SWCONFIG_SUPPORT_HW_NAT`.  Proc-fs interface for configuring WAN/LAN
VLAN IDs and NAT mapping rules.

The switch core can perform hardware NAT translation (SNAT/DNAT) at wire
speed by rewriting IP addresses and L4 ports in hardware.  This is what
enables a gigabit home gateway to route at line rate without CPU involvement.
A large feature requiring the full HNAT subsystem from the BSP.  Not currently
planned.

---

#### ACL offload (hardware firewall / filtering) — `[l2Driver]`

**Registers:** `MSCR` bits 3 = `MSCR_EN_IN_ACL`, 4 = `MSCR_EN_OUT_ACL`
(currently disabled).  ACL table via TACI type 5.

The switch core has an Access Control List engine for hardware packet
filtering.  The driver currently only reads ACL entry 0 as an 8198C workaround.
Enabling ACL offload would require programming TACI type 5 table entries.

---

#### IGMP/MLD snooping (hardware multicast) — `[l3Driver]`

**BSP subsystem:** `l3Driver/rtl865x_multicast.c`.  IP-level (S,G) lookup
tables in hardware.  The switch can snoop IGMP/MLD membership reports and
automatically route multicast streams only to subscribed ports.

This is separate from the bridge-mode multicast issue (`ndo_set_rx_mode`).
The hardware multicast table is for IP-layer multicast routing, not L2
promiscuous-mode forwarding.

---

#### EEE (802.3az Energy Efficient Ethernet) — `[AsicDriver]`

**Registers:** Extended PHY page writes to pages 32 (reg 22,23,26,27), 33
(reg 25,26), 40, 64, 65 (reg 21-23,28), 66 (reg 21) plus ram code patches.
BSP reference: `eee_phy_enable_98()`.

Requires per-port PHY register writes and N-way restart.  Green Ethernet
(power saving) is already enabled; EEE adds the 802.3az low-power idle mode.
Independent of gigabit — testable only with an EEE-capable link partner.

---

#### Giga Lite (500M mode) — `[AsicDriver]`

**Registers:** `EXTPCR0-5` bits `CF_HW_500M_AN` (bit 1), `CF_500M_EN` (bit 0);
`gphy_wb(0xa42, 20, ...)` for MAC-side and `Set_GPHYWB` page 0xa42/0xa4a for
PHY-side configuration.

A 500 Mbps half-step mode between 100M and 1000M.  Optional — not required
for full gigabit operation.

---

#### Bandwidth control / rate limiting (leaky bucket) — `[l2Driver]`

**Register:** `ILBPCR1` (PSCR block at 0xBB804800+).  Leaky bucket bounding
values for egress rate limiting.

Deferred because the PSCR register block hangs the bus if TRXRDY is not yet
set.  Our probe sets `SIRR_TRXRDY` after QoS init but the ILB write requires
additional investigation of the correct ordering.

---

#### Bridge mode (`ndo_set_rx_mode`) — `[Kernel Interface]`

**Registers:** `FFCR` bits 0 = `EN_UNUNICAST_TOCPU`, 1 = `EN_UNMCAST_TOCPU`
(currently both cleared).  Needed for promiscuous and allmulti mode under a
Linux bridge.

The BSP's `re865x_set_rx_mode()` is a stub (comments: "Not yet implemented").
No BSP reference to reuse.  Must be written from scratch: toggle FFCR TOCPU
bits based on `dev->flags & IFF_PROMISC` / `IFF_ALLMULTI`, and add FDB
entries for subscribed multicast addresses from `ndo_set_rx_mode`'s list.

---

#### Jumbo frames — `[AsicDriver]` + `[Kernel Interface]`

Requires descriptor and mbuf changes to support MTU > 1500.  The switch core
hardware supports jumbo frames but the current descriptor format uses fixed-size
cluster buffers.

---

#### PTP (1588 / 802.1AS hardware timestamping) — `[AsicDriver]`

**Fields:** `ph_ptpPkt`, `ph_ptpVer`, `ph_ptpMsgType` in the pkthdr struct
(3 reserved bytes in our struct, used by the BSP exclusively for 8198C).

The hardware can timestamp PTP packets at ingress and egress.  Requires
adding `ph_ptp*` fields to our pkthdr and implementing the timestamp
capture/readout path.

## Descriptor Format — Legacy pkthdr/mbuf (NOT flat 24-byte)

**RTL8198C uses the legacy pkthdr/mbuf descriptor format**, matching the
bootloader's `swNic_poll.c` and the kernel BSP's `swNic_init()` (not the
new `New_swNic_init()` from `rtl819x_swNic.c`).  `CONFIG_RTL_SWITCH_NEW_DESCRIPTOR`
is defined **only for 8197F** in the BSP (`rtl_types.h` line 871-873).

The bootloader's `swCore_init()` is compiled with **both `RTL8198` and
`CONFIG_RTL8881A` defined**, taking the RTL8881A path for table clearing
(MEMCR=0x7F, only Multicast+Netif tables cleared) and the RTL8198C path
for LED init (PIN_MUX_SEL3, LEDCR0).

### Legacy descriptor structures (32-byte pkthdr + 24-byte mbuf)

```
struct pkthdr {                     // 32 bytes (with padding)
    union { void *pkthdr_next; struct mbuf *mbuf_first; };  // +0, 4B
    u16 ph_len;                                              // +4, 2B
    u16 bitfields1;                                          // +6, 2B
    u16 bitfields2;                                          // +8, 2B
    u16 ph_reason;                                           // +10, 2B
    u16 ph_flags;                                            // +12, 2B
    u8  ph_orgtos;                                           // +14, 1B
    u8  ph_portlist;                                         // +15, 1B
    u16 ph_flags2;                                           // +16, 2B
    u8  ph_pad[12];                                          // +18, 12B
};

struct mbuf {                       // 24 bytes
    struct mbuf *m_next;            // +0, 4B
    struct pkthdr *m_pkthdr;        // +4, 4B
    u16 m_len;                      // +8, 2B
    u8  m_flags;                    // +10, 1B
    u8  m_reserved2;               // +11, 1B
    u8  *m_data;                    // +12, 4B
    u8  *m_extbuf;                  // +16, 4B
    u16 m_extsize;                  // +20, 2B
    u8  m_reserved[2];              // +22, 2B
};
```

### OWN bit convention (from bootloader `asicregs.h`)

| Constant | Value | Meaning |
|----------|-------|---------|
| DESC_OWNED_BIT | 0x0001 | Mask |
| DESC_RISC_OWNED | 0x0000 | bit0=0: RISC owns it (needs CPU action) |
| DESC_SWCORE_OWNED | 0x0001 | bit0=1: CPU/SWCORE owns it |
| DESC_WRAP | 0x0002 | Wrap bit |

**This is inverted from what you might expect!**  The bootloader defines
`DESC_RISC_OWNED = (0 << 0) = 0x0000` and `DESC_SWCORE_OWNED = (1 << 0) = 0x0001`.

### Ring format (legacy)

```
RX:  rxPkthdrRing[N] = array of u32, each entry = (KSEG1 ptr to pkthdr) | OWN|WRAP
     rxMbufRing[N]    = array of u32, each entry = (KSEG1 ptr to mbuf)  | OWN|WRAP
TX:  txPkthdrRing[N]  = array of u32, each entry = (KSEG1 ptr to pkthdr) | OWN|WRAP

Registers: CPURPDCR0-5 (RX), CPURMDCR0 (RX mbuf), CPUTPDCR0-3 (TX)
```

All pointers must be KSEG1 (0xAxxxxxxx+) — the RISC operates entirely in
uncached KSEG1 space.  Pools allocated via kmalloc, converted with kseg1().

### Flat 24-byte descriptor format (ALTERNATIVE — `#else` path)

The driver supports the flat 24-byte format as a compile-time alternative
(comment out `#define DESC_FORMAT_LEGACY`).  This path has its own DMA_CR0,
TXRINGCR, and CPUICR register writes.  It has the same issue as the legacy
format: the RISC won't deliver frames to the CPU.

## KSEG1 vs Physical Addresses

| What | Needs KSEG1? | Notes |
|------|-------------|-------|
| Register access (CPU-side) | Either | Both KSEG1 and physical work for CPU reads/writes |
| **SWTAA (TACI table address)** | **KSEG1 ONLY** | TACI engine runs inside RLX5281, expects KSEG1 |
| CPURPDCR0/CPUTPDCR0 (ring bases) | Either | Bootloader uses KSEG1, flat format uses physical — both work |
| Descriptor `addr` fields | Physical | Our driver uses physical for buffer addresses |
| `devmem` read/write | Physical | Busybox devmem expects physical, not KSEG1 |
| TCR0-7, TACR, TASR (TACI control) | Physical | These are CPU-facing registers |

**Rule:** If a value is consumed by CPU hardware → either format.  If it's
forwarded to a switch-core-internal engine (TACI) → KSEG1.

## Physical Memory Map

```
0x18000000  Physical base of SoC IO block
0xB8000000  KSEG1 virtual (= 0xA0000000 | 0x18000000)
```

## GMAC (0xB8012000) — NON-FUNCTIONAL

DMA register bank (offsets 0x1300+) reads back zero after writes.  MAC
registers (0x00-0xFF) work.  Path abandoned — use switch core CPU interface.

## TACI — Table Access Control Interface (0xBB804D00+)

| Offset | Name | Bits | Description |
|--------|------|------|-------------|
| 0x00 | SWTACR | bit0=ACTION_START, bit1=CMD_ADD, bit2=CMD_DEL, bit3=CMD_FORCE | Table Access Control |
| 0x04 | SWTASR | bits1:0=TABSTS | Table Access Status |
| 0x08 | SWTAA | | Table Access Address (**KSEG1 only!**) |
| 0x20+ | TCR0-7 | | Table Content Registers |

### Table base addresses

`address = 0xBB000000 + (table_type << 16) + entry_index * 32`

| Type | Base (KSEG1) | Description |
|------|-------------|-------------|
| 0 | 0xBB000000 | L2 switching table (**CMD_FORCE for writes**) |
| 5 | 0xBB050000 | External IP table |
| 6 | 0xBB060000 | VLAN table (**CMD_ADD for writes**) |

### L2 table write confirmed working

The L2 table IS writable via TACI — **SWTAA must use KSEG1 addresses**.
Previous failures were caused by using physical addresses in SWTAA.
CMD_FORCE (BIT(0)|BIT(3)=0x9) works; CMD_ADD (0x3) also works.

### L2 entry format (2 words, stored at 32-byte intervals)

```
Word 0: (mac[3]<<24) | (mac[4]<<16) | (mac[1]<<8) | mac[2]
Word 1: (mac[0]<<24) | (memberPort<<18) | (toCPU<<14) | (isStatic<<13)
        | (aging<<11) | (fid<<7) | (auth<<6)
```

### VLAN entry format (Word 0)

| Bits | Name | Description |
|------|------|-------------|
| 5:0 | memberPort | Physical ports 0–5 as members |
| 8:6 | extMemberPort | Extension ports (CPU = bit 0) |
| 14:9 | egressUntag | Untag on egress, ports 0–5 |
| 17:15 | extEgressUntag | Untag on egress, extension ports |

## Current Driver Init Sequence (probe)

```
switch_reset()              // Clock gate toggle only (no FULL_RST, matches bootloader)
  -> SYS_CLK_MAG &= ~CM_ACTIVE_SWCORE; delay; |= CM_ACTIVE_SWCORE

switch_l2_init()            // Minimal init matching bootloader
  -> MSCR disable all, ALECR L2_CLEAR, RMACR clear
  -> #if 0: MACCR, SWTCR0, VCR0, CSCR (bootloader doesn't write these)
  -> FFCR = EN_UNMCAST_TOCPU | EN_UNUNICAST_TOCPU
  -> SWTCR1: NAT/Frag/L4 enable
  -> l2_table_clear(): MEMCR=0x7F (bootloader value) + MIB clear
  -> #if 0: vlan_policy_setup, bandwidth_init, flow_control_init
  -> PCRP MacSwReset cycle ports 0-4, STP forwarding all ports
  -> PCRP6: STP forwarding + EnablePHYIf
  -> QNUMCR: 1 queue per port
  -> #if 0: MACCTRL1
  -> PVID = 8 for all ports (0-6)
  -> #if 0: qos_dscp_init, leaky_bucket_init, eee_disable, led_init
  -> VLAN VID=8: all ports+CPU, untagged (via TACI CMD_ADD)
  -> l2_fdb_init(): broadcast + CPU MAC -> TRAPCPU (via TACI CMD_FORCE)
  -> MSCR = EN_L2 only (0x01)

switch_gphy_init()          // Sram98C patches + 10pf MDI fix
N-way restart               // AFTER GPHY patches

switch_start()              // SIRR=TRXRDY, GIMR, CPUIIMR=0x0A, CPUIISR clear
                            // Note: CPUICR NOT written here

alloc_rings()               // Legacy pkthdr/mbuf pools + ring bases
  -> KSEG1 pointers for pkthdr, mbuf, cluster pools
  -> CPURPDCR0, CPUTPDCR0, CPURMDCR0 = KSEG1 addresses
  -> CPUICR = 0xC4000008 (TXCMD|RXCMD|BUSBURST_32W|MBUF_2048B)
```

### ndo_open order

```
refill_rx()                 // No-op for legacy (pools are static)
Reprogram: CPURPDCR0, CPUTPDCR0, CPURMDCR0
CPUICR = 0xC4000008         // Restore after ndo_stop cleared it
switch_start()              // CPUIISR clear, CPUIIMR=0x0A, SIRR=TRXRDY, GIMR
```

### ndo_stop order

```
CPUIIMR = 0; CPUICR = 0; GIMR clear; tx_clean; free_rx_skbs
```

## Known Code-Level Issues

- **Ring size** — 64 entries (minimum tested).  4-entry rings hang the
  board due to kernel buffer requirements.
- **Hardcoded MAC** — fallback to eth_random_addr if device tree
  `local-mac-address` is absent.  Should read from board EEPROM/NAND.
